# v0.3.0.0 Release Notes

## Summary

`v0.3.0.0` is the compiled-runtime architecture release. The editable atomized source now lives in
`atomics/skill/`, while `skill/` is the generated compiled Claude package root.

The release also clarifies render modes: clean default output remains prose-first, while compact DSL
and fuller audit renders are explicit opt-in modes.

## Major Changes

- Canonical atomized source moved to `atomics/skill/`.
- Compiled Claude runtime generated into `skill/`.
- Compiler/checker toolchain added under `tools/`.
- Five runtime bundles added.
- Seven selective omnibus bundles added.
- Compiled path-resolution map added as `skill/compiled-module-map.json`.
- Routing parity fixtures added under `tests/routing-fixtures/`.
- Sub-20 normal file-call target enforced by static call-budget checks.
- Recursive traversal governance added.
- Default / DSL / audit render modes clarified.
- Root `AGENTS.md` contributor guide added.

## Render Modes

- `/daee-epistemics` = clean default response with internal recursive governance.
- `/daee-epistemics:dsl` = compact diagnostic / lab-report render.
- `/daee-epistemics:audit` = fuller procedural audit render.

Default mode still runs State Refresh, held-route rechecks, and STOP / HOLD / RECURSE / PARTIAL
governance internally. It should not print a giant load ledger by default.

## What Is Preserved

- Original module IDs.
- YAML operative contracts.
- Diagnostic IR discipline.
- Routing precedence.
- P7 STOP / HOLD / RECURSE / PARTIAL discipline.
- Source-basis traceability.
- Atomized source authority.
- Recursive traversal through eligible live doors.

## What Is Not Claimed

- Exact prose parity.
- Live model behavioral equivalence beyond static routing-parity and static governance checks.

## Deprecated Normal Prompting Pattern

The old grim-reaper prompt is no longer required for normal use. Its useful discipline has been
internalized into skill governance. Use `/daee-epistemics:dsl` for compact lab-report output and
`/daee-epistemics:audit` for fuller procedural output.

## Remaining Risks

- Static parity is not live LLM behavioral parity.
- Future edits must regenerate `skill/`.
- Omnibus availability must not become activation.
- Default-vs-DSL-vs-audit render behavior still depends on model compliance.
