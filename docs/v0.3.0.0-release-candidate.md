## Render Mode Audit

Result: pass.

Default mode:

- Clean prose and readable formatting are the default.
- Plain `/daee-epistemics` does not request a lab report or giant load ledger.
- Recursive traversal, State Refresh, held-route recheck, and STOP / HOLD / RECURSE / PARTIAL governance still run internally.
- Follow-up prompts such as "continue" resume from the previous state-refresh boundary.

`:dsl` mode:

- `/daee-epistemics:dsl` is documented as compact diagnostic / lab-report mode.
- It may show compressed Case State or Diagnostic IR, live doors, matched modules by original module IDs, held routes, State Refresh, and STOP / HOLD / RECURSE / PARTIAL.
- It avoids the full verbose load ledger by default.

`:audit` mode:

- `/daee-epistemics:audit` is documented as fuller procedural audit mode.
- It may expose runtime/bundle ledger, source-basis, diagnostic IR gate, routing gate, render permission, recursive passes, state refreshes, and final governance.
- Any atomized path in audit mode must resolve through `compiled-module-map.json` when the standalone file is absent from the generated runtime.

Static checker added:

```text
tools/check_render_modes.py
```

It verifies generated runtime surfaces contain the canonical invocations, default render mode, Diagnostic IR, State Refresh, live door, held routes, STOP / HOLD / RECURSE / PARTIAL, no giant load ledger by default, bundle availability is not activation, compiled path resolution, and grim-reaper prompt deprecation.

## Grim-Reaper Prompt Deprecation

The old grim-reaper prompt is deprecated as a normal invocation pattern.

Its useful behavior is now native:

```text
mandatory state refresh
STOP must be earned
RECURSE when another eligible live door remains
HOLD only for absent release signal
PARTIAL when limits prevent completion
recursion is one door at a time, not argument dumping
```

Use `/daee-epistemics:dsl` for compact lab-report output. Use `/daee-epistemics:audit` for fuller procedural output closest to the old prompt's visible ledger.
