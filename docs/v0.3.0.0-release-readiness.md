# v0.3.0.0 Release Readiness

## Summary

Phase 8 audited the repository for the compiled-runtime architecture release.

Release posture: ready with caveats. The repo is consistent for `v0.3.0.0` static release readiness:
edit `atomics/skill/`, compile into `skill/`, run the checker suite, and package the contents of
`skill/`. Static routing parity and recursive traversal governance pass. Live model behavioral
equivalence and exact prose parity are not asserted.

## Layout Verification

Current layout:

```text
atomics/skill/          = canonical atomized editable source
skill/                  = generated compiled Claude package root
tools/                  = compiler/checker scripts
tests/routing-fixtures/ = static routing parity fixtures
docs/                   = architecture, audit, and workflow notes
build/                  = optional local release/package outputs
```

The generated runtime remains a build product. Generated files under `skill/` were rebuilt with
`tools/build_compiled_runtime.py`; no generated runtime file was hand-edited.

## Framework Pipeline Audit

Result: pass after updates.

The audited runtime route is:

```text
V1
-> Phase 2 passes
-> Diagnostic IR
-> routing precedence
-> dispatch gate
-> matched original module IDs
-> compiled bundle path-resolution
-> output-release governance
-> post-render state refresh
-> STOP / HOLD / RECURSE / PARTIAL
```

| Pipeline section | Current issue | Proposed update | Risk if not fixed |
|---|---|---|---|
| `atomics/skill/SKILL.md` minimum execution load floor | Matched modules were described as standalone files in the skill package, which could be misread in the compiled runtime. | Updated wording so matched modules are original module IDs/source identities resolved to source files in atomized mode or compiled bundle sections in runtime mode. | Claude or contributors could chase missing atomized files in `skill/` or treat missing files as gate failures. |
| `diagnostic-ir.md` ghost-load rule | Ghost-load language named only a loaded file, not the compiled section equivalent. | Updated the rule to cover source files and compiled runtime sections. | A bundle section could be loaded without source-basis traceability while still appearing compliant. |
| `framework-pipeline.md` compact pipeline | Pipeline did not explicitly state compiled path-resolution at the pipeline owner. | Added a compiled runtime note: atomized paths are source identities resolved through `compiled-module-map.json`; omnibus filenames are containers, not active dispatch. | Runtime readers could treat atomized paths as literal runtime file-load targets. |
| Routing precedence / output release / render / P7 | Core order and recursive governance were already current after Phase 6. | No architecture change required. Verified in generated runtime through the recursive checker. | Premature STOP, argument dumping, or permanently suppressed held routes. |
