## Summary

v0.3.0.0 converts daee-epistemics from an atomized runtime skill into a compiled low-call runtime while preserving the atomized source as canonical. It adds compiler/checker tooling, routing parity checks, recursive traversal governance, render modes, and release-ready Claude packaging.

The release also fixes a render-mode regression (Phase 10) in which the default `/daee-epistemics` invocation was emitting the full `[Diagnostic IR]` code-fenced schema block — `:audit`-mode behavior — in ordinary bounded prose responses.

Core invariant established: **Discipline is universal; printout is mode-specific.**

---

## What's New

### Compiled Runtime Architecture

- `atomics/skill/` is now the canonical editable atomized source. Edit this tree.
- `skill/` is now the generated compiled Claude package root. Do not hand-edit.
- `tools/build_compiled_runtime.py` compiles the runtime from atomized sources.
- Five always/near-always runtime bundles: `runtime-foundation`, `runtime-diagnostic-core`, `runtime-phase2-passes`, `runtime-dispatch-gate`, `runtime-output-governance`.
- Seven selective omnibus bundles for profiles, tactics, techniques, procedures, DO families, RT/transmission, and specialty diagnostics.
- `compiled-module-map.json` resolves original module IDs to compiled bundle paths.
- `build-manifest.json` records build metadata, bundle checksums, and source checksums.
- `tools/compiled_runtime_lib.py` provides shared compiler utilities.

### Render-Mode Governance

Three canonical render modes now formally governed:

| Mode | Level | IR Block | Load Ledger |
|------|-------|----------|-------------|
| `/daee-epistemics` | 1 — prose-first | NO | NO |
| `/daee-epistemics:dsl` | 2 — compact diagnostic | NO | NO |
| `/daee-epistemics:audit` | 3 — full procedural | Operative fields only | Permitted |

### Render-Mode Regression Fix

Root cause: `diagnostic-ir.md` Full IR Schema had no prohibition against default-mode printing. Fixed in three canonical atomics files: `diagnostic-ir.md`, `diagnostic-render-contract.md`, `SKILL.md`.

`check_render_modes.py` verifies 23 required invariant tokens, 5 forbidden claims, and 5 dispatch-gate policy tokens.

### Recursive Traversal Governance

- State Refresh after every bounded move — not only on user reply.
- No premature STOP while an eligible live door remains.
- HOLD / PARTIAL / RECURSE discipline enforced.
- Recursion is one door at a time.

## What's Not New

- Original module IDs are preserved throughout.
- YAML operative contracts (`contract_version` field values updated but schema unchanged).
- Diagnostic IR discipline is preserved and strengthened.
- Routing precedence is preserved.
- P7 STOP / HOLD / RECURSE / PARTIAL discipline is preserved.
- Source-basis traceability is preserved.
- Atomized source under `atomics/skill/` remains authoritative.
- Omnibus files are containers, not matched modules. Availability ≠ activation.
- Meta-noetic diagnosis still governs module release. DSL/IR still prevents omnibus argument dumping.
- Exact prose parity is not claimed.
- Live model behavioral equivalence is not claimed beyond static routing/render checks.

## Remaining Caveats

- Static checks do not prove live model behavioral equivalence.
- Future source edits must happen in `atomics/skill/` and regenerate `skill/` via `tools/build_compiled_runtime.py`.
- Omnibus availability must not become activation; live inference must comply.
- Default-vs-DSL-vs-audit render behavior depends on model compliance with the generated runtime instructions.
- Git operations (commit, push, tag) must be performed from the host machine — the build sandbox cannot access `.git`.
