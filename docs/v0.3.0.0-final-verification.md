## Render Mode Regression Fix

**Root cause:** `atomics/skill/references/diagnostics/diagnostic-ir.md` contained the Full IR Schema with no prohibition against printing it in default mode. The model treated the schema as an output template, causing the `## Diagnostic IR` section header and code-fenced block to appear in ordinary bounded responses (Level 3 / `:audit` behavior).

**Files changed in canonical atomics source:**

| File | Change |
|------|--------|
| `atomics/skill/references/diagnostics/diagnostic-ir.md` | Added `## Render-Mode Policy — IR as Internal State, Not Printout Template` section |
| `atomics/skill/references/rubrics/diagnostic-render-contract.md` | Added Prohibited Render Move #10 |
| `atomics/skill/SKILL.md` | Added "Default mode must not print" list in §V |

**Post-fix render behavior:**

| Mode | IR Block | Load Ledger | Case State | Post-Render Gate |
|------|----------|-------------|------------|-----------------|
| `/daee-epistemics` | NO | NO | Internal only | Compact governance sentence only |
| `/daee-epistemics:dsl` | NO | NO | Visible (compressed) | Full gate visible |
| `/daee-epistemics:audit` | Operative fields only | Permitted | Full | Full |

**Core invariant:** Discipline is universal; printout is mode-specific. The grim-reaper discipline applies in every mode; the grim-reaper printout belongs to `:audit`.

## Remaining Caveats

1. **Static checks only.** The suite verifies structural invariants. No live inference pass tests exist. Render-mode compliance and routing correctness depend on model behavior at runtime.
2. **No live behavioral equivalence test.** v0.2.x vs. v0.3.0.0 equivalence rests on static routing parity and structural checks only.
3. **virtiofs build environment.** Cannot delete files; workarounds (truncation, new filenames) used for build artifacts. Does not affect runtime content.
4. **Omnibus activation depends on model compliance.** Checker confirms omnibus filenames are not `matched_modules`; live inference must still respect this at invocation time.
5. **Git operations require host machine.** The sandbox cannot access `.git`. Commit and push must be done from Windows.
